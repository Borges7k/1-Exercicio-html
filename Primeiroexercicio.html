Como funciona a comunicação do Cliente x Server

(Voltar)
  1. Cliente e Servidor:Seria o cliente é o navegador web que você usa, como Chrome ou Opera, e o servidor é onde as páginas e os dados estão armazenados.
  2. Requisição:Quando você digita um URL no navegador e aperta Enter, o navegador envia uma requisição HTTP ao servidor.
  3. Processamento no Servidor:O servidor recebe essa requisição, processa o pedido e, se necessário, interage com um banco de dados para buscar as informações requisitadas.

    4. Resumindo, a comunicação entre cliente e servidor é um ciclo de requisições e respostas que possibilita a interação dinâmica na web. Essa estrutura é fundamental para o funcionamento de aplicações modernas.

O que eu aprendi sobre as TAG

(Voltar)
  1. A tag P é super útil para criar parágrafos no nosso texto. Quando a usamos, o navegador já organiza tudo pra gente, deixando um espaço em cima e embaixo do parágrafo. Assim, fica mais fácil de ler e a página
    não fica toda amontoadinha.
  2. As tags h1 até á h6 são para os títulos da nossa página. O h1 é o mais importante, como o título principal, e os outros vão descendo em importância até o h6. Isso ajuda a organizar o conteúdo em seções e subtítulos.
  3. A tag br que estou usando é usada para quebra de linha em texto. É ótima quando queremos que algo comece em uma nova linha, sem criar um novo parágrafo
  4. A tag title é importante porque define o título da nossa página. Ele aparece na aba do navegador e é uma das primeiras coisas que as pessoas veem.
  5. A tag a é usada para criar links. Com o atributo href, podemos definir para onde queremos que o link leve. Se você quiser abrir em uma nova aba, é só adicionar target="_blank".
    Essa tag é fundamental para navegar entre páginas ou até mesmo para âncoras, que são aqueles links que levam a uma parte específica da mesma página.

Listas ordenadas e Não ordenadas

(Voltar)

Quando falamos sobre listas em HTML, temos dois tipos principais: as listas ordenadas e as não ordenadas.

Listas Não Ordenadas ul

As listas não ordenadas são aquelas que usamos quando a ordem dos itens não importa. Por exemplo, se eu estiver listando frutas que eu gosto, eu posso colocar maçã, banana e laranja em qualquer ordem. Para criar uma lista não ordenada, usamos a tag ule dentro dela, colocamos cada item com a tag. O resultado aparece com marcadores, como bolinhas, o que ajuda a visualizar melhor a lista.

Listas Ordenadas ol

Por outro lado, temos as listas ordenadas, que usamos quando a sequência é importante. Um exemplo clássico é uma receita, onde os passos precisam ser seguidos em uma ordem específica. Para isso, utilizamos a tag ol e, novamente, usamos li para cada item. Neste caso, os itens aparecem numerados, o que deixa claro a ordem em que devem ser seguidos.

Resumindo, a escolha entre uma lista ordenada e uma não ordenada depende da necessidade de ordem ou não dos itens que estamos apresentando. Essa organização ajuda a tornar o conteúdo mais claro e fácil de entender, tanto para quem lê quanto para os motores de busca.
